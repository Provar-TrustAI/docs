# /doc-terminology-guard — the vocabulary law, mechanically enforced

`30942c1` (Datasets→Scenarios, 24 files) proved the method: grep every residual hit of the old
vocabulary and **classify each against an allowlist** instead of blind find-replace. That
discipline lived only in one commit message; this skill makes it a standing gate. `doc-validate`'s
style dimension is per-page and loop-bound — this is the cheap whole-repo sweep.

## The law (sync with `AGENTS.md` — that file is the authority, this skill is the detector)

| Pattern (case-insensitive unless noted) | Rule |
|---|---|
| `scenario set` | Retired noun (sets removed v2026.06.30.1). Allowed only in: `changelog.mdx` history entries, archive folders/tags, quotes of vestigial in-app strings explicitly framed as vestiges. |
| `dataset` | Wire-truth only: `/v1/datasets`, `dataset_id`, `dataset row` as *legacy API container* framing, and changelog history. Never as the surface/product noun. |
| `paddington` (any case) | Reader-visible prose: never. Allowed: wire-format keys in `api-reference/*.json` (e.g. `emit_paddington_sessions`) and `docs-plan/` internal material. |
| `/paddington/` or `/v1/internal` | Never anywhere published (including the OpenAPI files) — hard fail. |
| lucide `bot` icon (`icon: "bot"`, `icon="bot"`) | Never — Trust Agent surfaces use `sparkles`. |
| `uploaded` / `generated` / `simulated` / `synced` as a session *origin label* | Retired taxonomy. Source is exactly **Scenario / Playground / External**. (Verb uses — "sessions generated by the wizard" — are fine; judge the noun-label sense.) |
| `Ask mode`, `Ingest now`, `Add to dataset`, `New Run flow` | Retired UI labels; the current labels are Chat/Simulate, Sync now, Evaluate, the Evaluation Launcher. |
| `Routes` column on Sessions, `topics` | Sessions say **Routed To**; Scenarios say **Expected routes**. |
| `coming soon`, `roadmap` | Each hit must name something that is still actually unshipped at the current tag — verify against the release notes; a stale "(coming soon)" for a shipped thing is a finding. |
| `row` for a scenario | A scenario is an entity with an SCN id, not a row. |

## Procedure

1. Read the current `AGENTS.md` terminology + content-boundary blocks first — if this table and
   that file disagree, that file wins and THIS SKILL needs a PR.
2. Sweep published surfaces only: `*.mdx` at root + `concepts/ tutorials/ how-to/ snippets/` and
   `api-reference/*.json`. Exclude `docs-plan/`, `.claude/`, archives (`v20*/`), `README.md`,
   `CONTRIBUTING.md`.
   ```bash
   grep -rniE 'scenario set|paddington|"bot"|ask mode|ingest now|add to dataset|coming soon' \
     --include='*.mdx' --include='*.json' . | grep -vE '^\./(docs-plan|\.claude|v20)'
   ```
   (Split per-pattern in practice; the combined grep is illustrative.)
3. **Classify every hit** — `violation` (fix it), `allowlisted` (say which rule), or
   `law-is-stale` (the product changed again → the fix belongs in `AGENTS.md` first, then here).
4. Output a classified table. In gate mode (invoked from a landing), any unclassified or
   `violation` hit blocks the landing; fix and re-run.

## When to run

- As a `/doc-land-release` gate step (before the worktrunk→main merge).
- Immediately after any release that renames product vocabulary (pair with `/doc-release-align` —
  its terminology findings update `AGENTS.md`, then this sweep finds the stragglers).
- Standalone whenever prose "smells like last release".
