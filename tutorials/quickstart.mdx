This tutorial walks you from a brand-new Trust AI account to a curated [Scenario set](/concepts/scenarios) ready for evaluation, in roughly ten minutes. It assumes the **SI Consultant** persona: setting up Trust AI for a customer agent for the first time. The shape is the same for an Agent Owner adding a new agent to an existing workspace.

Most of these steps you'll drive by asking the [Trust Agent](/concepts/trust-agent) — the in-app assistant — in plain language, then watching the results land in the UI. Where you'd rather do something by hand, the manual UI path is right there too.

By the end you'll have:

- A [Project](/concepts/projects) wired to a deployed agent
- One ingested [Session](/concepts/sessions) you can read end-to-end
- An [Annotation](/concepts/annotations) on a single Turn
- A starter Scenario set with one curated scenario

That's the smallest complete pass through the first three steps of [the evaluation loop](/concepts/evaluation-loop). Once it works once, the rest of Trust AI is "do this same thing at scale."

<Tip>
  Prefer to work through the API directly? Every endpoint, request shape, and response shape is auto-rendered in the [API Reference](/api-reference), the authoritative source.
</Tip>

## Before you start

You'll need:

- A Trust AI account with **Admin** or **SI Consultant** role on at least one org
- An agent **deployed and reachable** — today that means a Bedrock AgentCore runtime alias in an AWS account you can create an IAM role in
- About 10 minutes

If your agent isn't on AgentCore yet, you can still walk this tutorial using a **demo Project** seeded with example Sessions (we'll flag that variant inline).

## The five steps

<Steps>
  <Step title="Create your first Project">
    A [Project](/concepts/projects) is the workspace boundary around one agent (or fleet) under evaluation. Members, runtime credentials, and every artifact downstream belong to it.

    Creating a Project is the one setup step you do before the Trust Agent comes into play — the agent lives *inside* a Project, so you need one first. Click **+ New Project** in the top bar, name it after the agent under evaluation (e.g., "Acme Support Agent"), pick the org it belongs to, and click **Create**.

    **You should see:** the Project's empty home view, with five empty surfaces (Sessions, Annotations, Scenarios, Evaluators, Evaluations) and a banner prompting you to connect a runtime.
  </Step>

  <Step title="Connect an agent runtime">
    The supported connector today is **AWS Bedrock AgentCore**. The setup is a one-time IAM role grant; once it's in place, Sessions flow automatically.

    From the Project home, click **Connect runtime → AWS Bedrock AgentCore**. Trust AI walks you through:

    1. Generating a CloudFormation snippet that creates a cross-account IAM role
    2. Deploying it in your AWS account
    3. Pasting the role ARN back into Trust AI

    Full step-by-step (with the CloudFormation snippet inline): [Connect an AWS Bedrock AgentCore agent](/how-to/connect-agentcore).

    **You should see:** Project home now shows the runtime as **Connected**, with the AgentCore agent name + alias pinned to the Project's metadata.

    <Tip>
      **Don't have AgentCore handy?** Skip to a demo Project: from your org's home, click **+ New Project → Use demo data**. You'll get a Project pre-seeded with ~50 example Sessions across two Agent Versions. The rest of the tutorial works the same way against the demo data.
    </Tip>
  </Step>

  <Step title="Ingest your first Sessions">
    Once the runtime is connected, ingestion is automatic. Trust AI pulls AgentCore invocation logs every few minutes and each invocation becomes a Session row.

    <Note>
      *Ask the Trust Agent:* "Pull my latest sessions and show me the ten most recent."
    </Note>

    The agent runs the pull for you and renders the Sessions it found as a clickable table you can read right there. You'll see Sessions begin to appear within ~30 seconds for an active agent; or do it by hand from the **Sessions** tab with **Sync Now**.

    **You should see:** the Sessions list populated with rows. Each row shows: timestamp, Agent Version, turn count, latency, and any intent classification your agent emits.
  </Step>

  <Step title="Review and annotate a Session">
    Open the first Session in your list. Read through the Turns. The viewer shows:

    - The full conversation, top to bottom
    - Tool Calls inline (request + response, latency)
    - Skill activations as colored badges
    - A side panel where Annotations live

    Pick a single Turn — say, the one where the agent responded to the user's primary question.

    <Note>
      *Ask the Trust Agent:* "On the order-status session, mark the turn where the agent answered the customer as a pass, and note that it resolved order status and explained next steps clearly."
    </Note>

    Because adding an Annotation is a write, the Trust Agent pauses and shows you an approval card with the exact label and note before it saves — approve it and the Annotation lands. Prefer to do it by hand? Apply a pass/fail [Annotation](/concepts/annotations) directly on the Turn with a short freeform note, then click **Save**.

    **You should see:** the Annotation appear in the Session's side panel, time-stamped, attributed to you. The Sessions list now shows this row as **Annotated**.
  </Step>

  <Step title="Curate your first Scenario set">
    Now turn that one annotated Session into the first scenario in a curated [Scenario set](/concepts/scenarios).

    <Note>
      *Ask the Trust Agent:* "Create a new scenario set called 'Order-status happy path' and add that annotated session to it as the first scenario."
    </Note>

    The agent drafts the new Scenario set and the scenario, then pauses for your approval before it writes either — confirm, and both land. Or do it by hand: from the Sessions list, use the add-to-scenario-set action on your annotated Session and create a new set — it starts as "Untitled scenario set"; rename it (e.g., "Order-status happy path") and confirm.

    **You should see:** the Scenarios surface now has one Scenario set with one scenario. Open it — the scenario carries the Session's input, the agent's full response, and the Annotation you applied.
  </Step>
</Steps>

## You're done with the first half of the loop

You've done the first three steps of [the evaluation loop](/concepts/evaluation-loop) — ingest, annotate, curate. The next three are **define an Evaluator**, **run an Evaluation**, and **compare Agent Versions**. The Scenario set you just curated is the test bed they all run against. You can continue click by click with the next two tutorials, or let the Trust Agent drive the rest of the loop by conversation in [Evaluate with the Trust Agent](/tutorials/evaluate-with-the-trust-agent).
