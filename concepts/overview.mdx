Trust AI is built around seven core nouns. Every workflow, every API call, every screen in the product traces back to one of them. Read this page first, then dig into each noun's dedicated page from the cards below.

## The seven nouns

<CardGroup cols={2}>
  <Card title="Project" icon="sliders-horizontal" href="/concepts/projects">
    The workspace container вҖ” owns members, runtime connection config, and every Session, Dataset, Evaluator, Evaluation, and Agent Version under it.
  </Card>
  <Card title="Session" icon="message-square" href="/concepts/sessions">
    One real interaction between a user and an agent вҖ” a sequence of Turns and Tool Calls. The raw material every Dataset and Evaluation builds on.
  </Card>
  <Card title="Annotation" icon="tag">
    A human-authored pass/fail, rubric score, or freeform comment on a Turn or Session. How reviewer judgment becomes data that Evaluators can calibrate against.
  </Card>
  <Card title="Dataset" icon="database" href="/concepts/datasets">
    A curated, versioned collection of Sessions or Turns. The stable test bed every Evaluation runs against, snapshotted so historical results stay reproducible.
  </Card>
  <Card title="Evaluator" icon="wand-sparkles" href="/concepts/evaluators">
    A reusable judgment function вҖ” LLM Judge (plain-English criteria) or Structured Match (deterministic checks). Defined once, reused across many Evaluations.
  </Card>
  <Card title="Evaluation" icon="chart-line" href="/concepts/evaluations">
    A single run of one or more Evaluators across a Dataset, scoped to an Agent Version. Scored, comparable, reviewable вҖ” the artifact regressions get caught in.
  </Card>
  <Card title="Agent Version" icon="git-branch">
    A snapshot of the agent under test вҖ” prompt, model, tool set, deployment. Comparing two Agent Versions on the same Dataset is the core regression-testing loop.
  </Card>
</CardGroup>

All of this happens **inside a Project**. The Project is the boundary around the loop вҖ” members, runtime credentials, and every artifact above belong to one specific Project. A Trust AI organization may have many Projects (one per agent or agent fleet under evaluation), but a single workflow always stays inside one.

## The two nouns without their own page (yet)

Annotation and Agent Version currently appear in the cards above but link to nothing вҖ” they don't have dedicated concept pages in this version of the docs. The shapes are real and stable in the product; the standalone reference pages are next on the M2 content backlog.

**Annotation in one paragraph:** a human-authored signal вҖ” pass/fail label, rubric score, or freeform comment вҖ” attached to a Turn or Session. Annotations flow into the calibration loop for Evaluators: a human annotates a few representative examples, then an Evaluator (especially an LLM Judge) gets refined until it agrees with the human pattern.

**Agent Version in one paragraph:** a snapshot of everything that defines an agent at a moment in time вҖ” prompt template, model identity, tool set, deployment URL. Every Evaluation result is scoped to one Agent Version. Comparing the same Dataset under the same Evaluators across two Agent Versions is how Trust AI surfaces regressions before they reach customers.
