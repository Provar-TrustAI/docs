The vocabulary of Trust AI. Icons match the Trust AI app's navigation, so the concept you read about here is the same concept you'll click on in product. Cards link through to dedicated concept pages where they exist.

## Core entities

<CardGroup cols={2}>
  <Card title="Project" icon="sliders-horizontal" href="/concepts/projects">
    The top-level workspace container. Owns members, runtime connection config (AgentCore, Salesforce Agentforce, and similar), and every Session, Dataset, Evaluator, Evaluation, and Agent Version under it.
  </Card>
  <Card title="Session" icon="message-square" href="/concepts/sessions">
    One real interaction between a user and an agent — a sequence of Turns and Tool Calls captured from production or staging. The raw material every downstream Dataset and Evaluation builds on.
  </Card>
  <Card title="Annotation" icon="tag" href="/concepts/annotations">
    A human-authored signal attached to a Turn or Session — a pass/fail label, rubric score, or freeform note. Encodes reviewer judgment into data that Evaluators can be calibrated against.
  </Card>
  <Card title="Dataset" icon="database" href="/concepts/datasets">
    A curated, versioned collection of Sessions or Turns chosen as a representative test bed. Rows come from ingested Sessions, manual entries, or CSV uploads. The stable substrate every Evaluation runs against.
  </Card>
  <Card title="Evaluator" icon="wand-sparkles" href="/concepts/evaluators">
    A reusable judgment function — LLM Judge (plain-English criteria sent to Claude or GPT) or Structured Match (deterministic checks on agent output metadata) — that scores a Turn or Session against criteria.
  </Card>
  <Card title="Evaluation" icon="chart-line" href="/concepts/evaluations">
    A single run of one or more Evaluators across a Dataset, scoped to a specific Agent Version. The output is a scored, comparable, reviewable record per Dataset row.
  </Card>
  <Card title="Agent Version" icon="git-branch" href="/concepts/agent-versions">
    A snapshot of the agent under test — prompt, model, tool set, deployment. Comparing two Agent Versions on the same Dataset under the same Evaluators is the core regression-testing loop.
  </Card>
</CardGroup>

## Session internals

<CardGroup cols={2}>
  <Card title="Turn" icon="corner-down-right">
    One message exchange inside a Session — the user message and the agent's response. The smallest unit an Evaluator can score.
  </Card>
  <Card title="Tool Call" icon="terminal">
    An invocation of an agent tool inside a Turn — function name, arguments, and the returned value. Structured Match Evaluators commonly assert on these.
  </Card>
</CardGroup>

## UI patterns

<CardGroup cols={2}>
  <Card title="Saved View" icon="bookmark" href="/concepts/saved-views">
    A named, persisted combination of filters, sort order, and column configuration for a list surface. Save a lens on your data and share it with your team.
  </Card>
  <Card title="Fly-in panel" icon="panel-right">
    A slide-out detail panel that opens from the right when you click a row in Sessions, Datasets, or Evaluations. Shows the full transcript, metadata, and contextual actions without leaving the list.
  </Card>
  <Card title="Verdict chip" icon="badge-check">
    A color-coded pass/fail or score badge on the Evaluation results matrix. Each evaluator applied to a dataset row produces one verdict chip. Green = pass, red = fail, numeric = rubric score.
  </Card>
  <Card title="Results matrix" icon="grid-3x3">
    The grid view on the Evaluation Run Detail page. Rows are dataset entries, columns are evaluators. Each cell is a verdict chip. The matrix is the at-a-glance view of how an agent version performed.
  </Card>
</CardGroup>

## Integrations

<CardGroup cols={2}>
  <Card title="MCP Server" icon="cable" href="/concepts/mcp-integration">
    The Model Context Protocol endpoint that lets external AI agents and IDEs read and write Trust AI data. Authenticated via WorkOS; write operations require approval.
  </Card>
</CardGroup>

## Paddington (agent assistant)

<CardGroup cols={2}>
  <Card title="Skill" icon="puzzle">
    A reusable capability the Paddington assistant can invoke inside Trust AI — for example, drafting an LLM Judge rubric or summarizing Sessions that match a filter.
  </Card>
  <Card title="Task" icon="list-checks">
    A unit of work Paddington plans and executes by composing Skills. How the assistant gets things done without you leaving the product.
  </Card>
</CardGroup>
