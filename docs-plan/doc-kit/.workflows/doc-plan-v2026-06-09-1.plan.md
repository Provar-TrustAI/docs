## 3. Hero-demo plan — the demo budget table + the wow-vs-overload resolution

### 3a. heroDemoGuidance (front and center — the resolution of Brady's CRITICAL concern)

**The rule:** the full end-to-end agent reel is shown *exactly once*, never on a page where someone arrives for a single fact. It lives only on `tutorials/evaluate-with-the-trust-agent.mdx`, and even there it is **the last element, collapsed in an accordion, click-to-play with native pause/replay (never autoplay)**. Each tutorial step first teaches one beat with its own short single-concept clip cut from that same master recording; the full reel is the opt-in "now watch it all at once" payoff. Why a recording and not a live embed of the reel: the shipped autoplay transport has only play/pause/back-to-start/skip-to-end (no scrubber) and is coupled to live data panels, so a faithful live port would drag half the app along — rendering it once as an MP4 removes both problems and one render produces both the hero reel and every per-step clip. The per-page wow instead comes from three genuinely self-contained live prototypes (each does exactly one thing, visible-on-arrival on its own page) plus one Tier-2 simulate walkthrough. **Net: no concept/how-to page asks a reader to absorb more than one moving idea; four heavy pages carry zero moving artifact; the only multi-concept block on the site is the capstone reel — opt-in, last, pausable.**
