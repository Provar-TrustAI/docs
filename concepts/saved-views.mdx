A Saved View is a named, persisted combination of filters, sort order, and column configuration for a list surface. Instead of re-applying the same filters every time you open Sessions or Datasets, save the configuration once and return to it with a click.

## Why Saved Views exist

Every list surface in Trust AI — Sessions, Datasets, Evaluations — supports filtering, sorting, and column selection. Teams quickly develop recurring lenses: "sessions from the last 24 hours with failed annotations," "the regression dataset sorted by tag," "evaluation runs triggered by CI." Without Saved Views, each team member rebuilds these filters manually every session.

A Saved View captures the filter state so the team shares a common lens on the data.

## What's in a Saved View

- **Filters** — which rows to include (by tag, status, time window, annotation state, etc.)
- **Sort order** — which column to sort by and in which direction
- **Column configuration** — which columns are visible and in what order
- **Surface binding** — which list surface the view applies to (Sessions, Datasets, or Evaluations)

## Default views

Any Saved View can be marked as your default for a given surface. When you open that surface, it loads with your default view's filters pre-applied instead of the unfiltered list. Defaults are per-user and per-project.

## In the app

- **View picker** — a dropdown in the toolbar of each list surface lets you switch between saved views or create a new one.
- **Save current view** — configure your filters and sort, then save the configuration with a name.
- **Set as default** — mark a view as your default so the surface opens with it pre-applied.

You can also manage Saved Views by asking the Trust Agent — for example, "save my current Sessions filters as a view called Failed last 24h," or "make that my default view." Saved Views are created, updated, and removed directly from the toolbar of each list surface, with default selection available per surface *(v2026.05.19+)*. Developers who need the underlying endpoint detail can find the Saved Views family in the [API Reference](/api-reference).
